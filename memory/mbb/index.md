# Memory Bank Bible - Index

Status: Draft (2026-03-26)

This section governs the memory bank itself. It defines the local rule set for structure, ownership, indexing, and authoring.

- [Principles](principles.md): The governing rules behind the system: single source of truth, one file per concept, annotated links, index-first navigation, and progressive disclosure. Read this first so the structure does not decay into duplicate or orphaned documents.
- [Structure](structure.md): The recommended folder layout, what each top-level section is for, and how indexes should work. Read this when bootstrapping a new section or deciding where a document belongs.
- [Artifact Types](artifacts.md): A map of the main document types, what question each one answers, when to create it, and what it should contain. Read this before adding a new file so you choose the right container.
- [Writing Format](format.md): The expected markdown shape, naming patterns, annotation rules, and how to split large topics. Read this while authoring so documents stay consistent and easy to scan.
- [Workflows](workflows.md): The practical operating model for creating, updating, and maintaining the memory bank during feature work. Read this when you want a repeatable process instead of ad hoc documentation.
- [Examples](examples.md): Reusable English examples and templates for indexes, specs, ADRs, feature plans, seeds, and scenarios. Read this when you want a concrete starting point rather than a rule description.
